为什么请求体都是使用流来读取的？

因为报文很长，请求航和请求头都在一个TCP报文中，都到数据后可以立刻读取，但是请求体可能被分割为很多的报文，在网络上慢悠悠的传输，所以必须用流的方式来读取。

如果不想麻烦，想一次性读取，可以使用一些框架，SpringMVC，koa的中间件。。。
